외래키(foreign key)

* foreign key 설정할 때 참조되는 테이블 
컬럼은 반드시 기본키(primary key)여야 합니다.*

제약조건 4가지

primary key 제약 조건
테이블에 존재하는 많은 행의 데이터를 구분할 수 있는 식별자를 'primary key(기본 키)' 라고 부른다.

foreign key 제약 조건
외래 키 제약 조건은 두 테이블 사이의 관계를 선언함으로써 데이터의 무결성을 보장해 주는 역할을 한다. 
외래 키 관계를 설정하면 하나의 테이블이 다른 테이블에 의존한다.
외래 키 테이블이 참조하는 기준 테이블의 열은 반드시 primary key 이거나 unique 제약 조건이 설정되어 있어야 한다.

unique 제약 조건
unique 제약 조건은'중복되지 않는 유일한 값'을 입력해야 하는 조건이다.
primary key와 거의 비슷하며 차이점은 unique는 null값을 허용한다는 것이다.

not null 값 허용
null 값을 허용하려면 null을, 사용하지 않으려면 not null을 사용하면된다.
prinary key가 설정된 열에는 null값이 있을 수 없으므로 생략하면 자동으로 not null로 인식한다.

MySQL LIKE 연산자

% : 모든 문자(글자수 제한x)를 인식한다.

_ : 문자 한글자

ENAME LIKE 'S%' -- S로 시작하는 모든문자를 인식
ENAME LIKE 'S_' -- S로시작하는 문자(총2글자)인식

ENAME LIKE '%S' -- S로 끝나는 모든문자를 인식
ENAME LIKE '_S' -- S로 끝나는 문자(총2글자)인식

ENAME LIKE '%S%' -- S라는 문자가 포함된 문자를 인식
ENAME LIKE '_S_' -- S라는 문자가 포함된 문자를 인식(총3글자)

mysql join

즉 서로 다른테이블에서 데이터를 가져올 때 사용하는 것이 join입니다.
